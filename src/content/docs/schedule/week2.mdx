### Software Engineering Topics

#### üåü SE Introduction

<details>
<summary><strong>SE: Intro</strong></summary>

<div style={{padding: '1rem', background: '#f8f9fa', borderRadius: '8px', marginTop: '1rem'}}>

**Software Engineering** is not just about writing code. It includes:
- Understanding requirements
- Designing solutions
- Managing projects
- Testing thoroughly
- Maintaining code over time

**Pros of Software Engineering practices:**
- Better code quality
- Easier maintenance
- Reduced bugs
- Team collaboration

**Cons:**
- Initial overhead
- Learning curve
- May seem unnecessary for small projects

</div>
</details>

#### üìä Project Management

<details>
<summary><strong>SDLC Process Models: Basics</strong></summary>

<div style={{padding: '1rem', background: '#f8f9fa', borderRadius: '8px', marginTop: '1rem'}}>

**What are SDLC process models?**
Software Development Life Cycle (SDLC) models provide structured approaches to software development.

**Sequential Models (e.g., Waterfall):**
- Complete each phase before moving to the next
- Good for projects with clear, stable requirements
- Hard to accommodate changes

**Iterative Models (e.g., Agile):**
- Develop in cycles/iterations
- Get feedback early and often
- Easier to accommodate changes

</div>
</details>

#### üîÑ Revision Control

<details>
<summary><strong>RCS: Revision History</strong></summary>

<div style={{padding: '1rem', background: '#f8f9fa', borderRadius: '8px', marginTop: '1rem'}}>

**Recording folder history:**
- Track changes over time
- See who made what changes
- Revert to previous versions if needed

**Backing up on cloud:**
- Push to remote repositories (GitHub)
- Protects against local failures
- Enables collaboration

**Using revision history:**
- `git log` - view commit history
- `git diff` - see changes
- `git checkout` - go to specific version

</div>
</details>

#### üíª Implementation

<details>
<summary><strong>IDEs: Basic Features</strong></summary>

<div style={{padding: '1rem', background: '#f8f9fa', borderRadius: '8px', marginTop: '1rem'}}>

**What are IDEs?**
Integrated Development Environments provide tools for coding in one place.

**Key features:**
- Syntax highlighting
- Auto-completion
- Error detection
- Debugging tools
- Version control integration

**IntelliJ IDEA setup:**
1. Create new project
2. Set up JDK
3. Configure project structure
4. Add libraries/dependencies

</div>
</details>

#### üß™ Testing

<details>
<summary><strong>Automated Testing of Text UIs</strong></summary>

<div style={{padding: '1rem', background: '#f8f9fa', borderRadius: '8px', marginTop: '1rem'}}>

**What is testing?**
Verifying that software works as expected.

**Regression Testing:**
- Re-run tests after changes
- Ensure new code doesn't break existing features

**Test Automation:**
- Write scripts to test automatically
- Faster than manual testing
- Consistent and repeatable

**Testing Text UIs:**
- Capture expected output
- Compare with actual output
- Use input/output redirection

</div>
</details>
